# Changelog

## Scopo del Documento
Questo file funge da registro storico e cronologico dell'evoluzione del software. Permette a sviluppatori, project manager e stakeholder di tracciare in modo trasparente ogni singola modifica apportata alla codebase versione dopo versione.

## Formato del Documento
Le modifiche verranno elencate per data e numero di versione (es. `v1.0.0`), suddivise rigorosamente in queste categorie:
- `Added`: Per le nuove funzionalità introdotte.
- `Changed`: Per le modifiche strutturali a funzionalità già esistenti.
- `Deprecated`: Per le funzionalità che verranno rimosse nelle prossime release.
- `Removed`: Per le funzionalità eliminate definitivamente.
- `Fixed`: Per i bug e i problemi tecnici risolti.
- `Security`: Per le patch o gli aggiornamenti relativi alla sicurezza.
