# Contributing Guidelines

## Scopo del Documento
Questo documento è il manuale operativo per la collaborazione del team sul repository. Il suo scopo è definire un flusso di lavoro (Git Workflow) chiaro e privo di conflitti, proteggendo il branch principale (`main`) da modifiche accidentali o non revisionate.

## Contenuto del Documento
In questo file verranno definiti:
- **Branching Strategy:** Le istruzioni su come creare e nominare correttamente i rami di lavoro.
- **Workflow di Sviluppo:** I passaggi esatti da seguire per sincronizzare il codice locale prima di effettuare un push.
- **Pull Request (PR) Process:** Le regole operative per aprire una PR, richiedere una Code Review ai colleghi e i criteri di approvazione necessari per il merge.
