# System Architecture

## Scopo del Documento
Questo documento fornisce la mappa strutturale e logica dell'intera applicazione. È fondamentale sia per l'onboarding di nuovi sviluppatori, sia per orientare correttamente gli strumenti di *vibecoding*, indicando l'esatta collocazione dei moduli e il funzionamento del "motore" software.

## Contenuto del Documento
In questo file verranno definiti:
- **Alberatura del Progetto (Directory Structure):** La spiegazione logica di come sono organizzate le directory e i moduli del repository.
- **Design di Sistema:** La separazione delle responsabilità e l'interazione tra i vari layer (es. Frontend, Backend, API).
- **Data Model:** Lo schema del database, le entità principali e le loro relazioni logiche.
- **Data Flow:** Il flusso di base delle informazioni e il ciclo di vita delle richieste all'interno del sistema.
